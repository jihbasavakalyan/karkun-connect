# User Guide — Karkun Connect 1.0 RC1

Karkun Connect supports two roles: **Administrator** and **Rukn**. This guide covers shared concepts and points to role-specific guides.

## Terminology

| Term | Meaning |
|------|---------|
| **Rukn** | Campaign supervisor responsible for assigned Karkuns |
| **Karkun** | Outreach volunteer assigned to a Rukn |
| **Campaign** | The active outreach period with defined start/end |
| **Annexure-1** | Visit record form completed during Karkun outreach |
| **Compliance** | Ijtema, JIH registration, monthly reporting, and Bait-ul-Maal tracking |

## Getting Started

1. Open the application URL provided by your administrator.
2. Sign in with your email and password.
3. Check **Remember Me** if you want to stay signed in on this device.
4. You are routed to the Administrator or Rukn portal based on your role.

## Administrator Workflow

See [Administrator Guide](administrator-guide.md) for the full campaign lifecycle:

Campaign → Rukn/Karkun masters → Assignments → Execution → Compliance → Follow-up → Reports

## Rukn Workflow

See [Rukn Guide](rukn-guide.md) for the field workflow:

Dashboard → My Karkun → Annexure-1 → Follow-up → Completion

## Session Behavior

- **Remember Me checked:** Session persists in browser local storage across restarts.
- **Remember Me unchecked:** Session lasts for the current browser tab only.
- **Logout:** Use the logout control in the top-right of Admin or Rukn layouts.

## Mobile Use

The Rukn portal is optimized for mobile field use. Bottom navigation provides quick access to Dashboard, My Karkun, and Campaign Record. Touch targets meet minimum 44px height.

## Data Notes (RC1)

All data in RC1 is stored in browser memory. A full page reload resets assignments and execution progress unless you have an active saved session. Production master data (Rukns and Karkuns) reloads automatically on each visit.
