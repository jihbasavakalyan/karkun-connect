# Documentation

Central repository for all project documentation related to **karkun-connect**.

This folder organizes product, technical, and design artifacts that guide development, onboarding, and long-term maintenance. Each subfolder covers a specific documentation domain.

## Subfolders

| Folder     | Purpose                                      |
| ---------- | -------------------------------------------- |
| `concept/` | High-level vision, goals, and domain context |
| `prd/`     | Product requirements and feature specifications |
| `srs/`     | Software requirements and system behavior    |
| `ui-ux/`   | Design system, wireframes, and UX guidelines |
| `database/` | Schema design, ERDs, and data conventions   |
| `testing/` | Test strategy, plans, and quality standards  |
